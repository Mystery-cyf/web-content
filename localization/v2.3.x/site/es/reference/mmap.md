      </svg>
    </button></h2><p>MMap files present a mechanism to map a shared memory region to a file. Leveraging MMap files, Milvus efficiently manages memory by loading collections onto the QueryNode’s disk instead of directly into memory. When a query is received, relevant data is dynamically loaded into memory, while unnecessary data is released, optimizing memory usage. Importantly, the immutability of data on QueryNodes ensures that removing data from memory doesn’t involve any disk write operations, preserving data integrity.</p>
<p>Based on an internal test, the Milvus instance with MMap enabled demonstrates robust performance when the volume of stored data remains constant. However, when the data volume increases to twice the original size, there is a notable 30% performance degradation. Nevertheless, MMap allows efficient storage, enabling up to four times the original data volume to be saved without a significant impact on performance.</p>

Consequently, MMap-enabled data storage is best suited for scenarios where performance is not a critical factor.</p>

      </svg>
    </button></h2><p>To enable MMap on Milvus, you need to change the configuration file as follows:</p>
<pre><code translate="no" class="language-yaml"><span class="hljs-comment"># new-values.yaml</span>
queryNode:
  mmapDirPath: <span class="hljs-built_in">any</span>/valid/path
<button class="copy-code-btn"></button></code></pre>
<p>After saving the file, apply the changes with the following command:</p>
<pre><code translate="no" class="language-shell">helm upgrade &lt;milvus-release&gt; --reuse-values -f <span class="hljs-keyword">new</span>-values.<span class="hljs-property">yaml</span> milvus/milvus
<button class="copy-code-btn"></button></code></pre>
<div class="alert notes">
<ul>
<li>MMap is activated when the specified path in <code translate="no">queryNode.mmapDirPath</code> is valid. To disable MMap, remove this setting item or leave it empty.</li>
<li>QueryNode utilizes its volume, rather than memory, for storing data loaded from collections.</li>
<li>Each scalar column in a loaded segment is stored in an individual file on the disk.</li>
<li>The index file of the vector field is consolidated into a single file on the disk.</li>
</ul>
</div>
